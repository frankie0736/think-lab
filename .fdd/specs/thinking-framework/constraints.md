# Scope 与约束

## In Scope（MVP）

### 功能
- [x] 6 阶段完整循环
- [x] 明确的阶段导航（用户知道在哪）
- [x] 统一使用 interview tool 交互
- [x] AI 自行判断阶段转换（无需用户确认）
- [x] 最终输出 Markdown 文档

### 文档结构
- [x] 问题定义（界定阶段输出）
- [x] 完整思维记录（每阶段的探索和取舍）
- [x] 结论（收敛阶段输出）
- [x] 反思（用户最后回答）

## Out of Scope（不做）

| 功能 | 理由 |
|------|------|
| 回溯机制 | 增加复杂度，单向流程更简单 |
| 领域特化 | 目标是"真正通用" |
| 多 Prompt 切换 | 保持简单，单个 System Prompt |
| 用户可配置深度 | AI 自动判断，降低用户心智负担 |
| 专业建议（法律/医疗） | 超出工具定位 |
| 保存历史会话 | MVP 不做 |

## Non-Goals（明确不做）

1. **不代替用户做决策** - 只帮用户"想清楚"，最终决策权在用户
2. **不强制完整流程** - 用户说"够了"就停，即使 AI 觉得不够深
3. **不过度工程化** - 简单问题也走流程，但每步更快

## 关键取舍

| 取舍 | 选择 | 放弃 | 理由 |
|------|------|------|------|
| 交互形式 | 统一 interview tool | 自由对话 | 结构化降低认知负担 |
| 阶段感知 | 明确导航 | 透明运作 | 用户需要知道"在哪" |
| 问题数量 | 固定 1-4 个 | AI 动态决定 | 保持可预期性 |
| 简单问题 | 走完整流程 | 快速通道 | 保持一致性 |
| 反思阶段 | 作为结束问题 | 省略 | 增加"元认知"价值 |
