# 范围与约束

## MVP Scope

### 必须做

1. **Post Type 设计引导**
   - 何时需要新建 Post Type
   - 区分产品 vs 其他内容类型

2. **Taxonomy 设计引导**
   - 分类法 vs 具体分类项
   - 分类维度规划（避免混乱）

3. **Field Group 规划**
   - 字段名、类型、说明
   - 通用字段 vs 特有字段
   - 买家视角的信息优先级

4. **Repeater 使用引导**
   - 何时使用（规格表、多图等）
   - 子字段设计

5. **结构化文档输出**
   - 清晰的 ACF 规划文档
   - 可喂给 ACF JSON 生成器

## Non-Goals

| 明确不做 | 原因 |
|----------|------|
| 前端展示设计 | 只管数据结构，不管呈现 |
| 业务流程 | 不涉及询价、下单等 |
| 多站点规划 | 只针对单个网站 |
| SEO/营销 | 不涉及关键词、营销策略 |
| Flexible Content 引导 | 太复杂，学员难掌握 |
| 行业预设模板 | 纯引导式，不预设行业 |
| 中间保存/进度恢复 | 一次性完成 |
| ACF JSON 直接生成 | 用户有独立的 JSON 生成器 |
| 复杂内容布局设计 | 文字+图+视频交替等展示布局不在范围 |

## 用户约束

- 目标用户：外贸学员（不懂技术）
- 使用场景：教学辅助
- 产品线复杂度：全覆盖（单品类多规格 / 多品类少规格 / 多层级组合 / 定制为主）
