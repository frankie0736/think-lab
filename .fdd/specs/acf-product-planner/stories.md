# 用户故事

## 核心用户

外贸学员：不懂技术，要建 B2B 产品展示站，不知道如何规划产品数据结构。

## 痛点

| 痛点 | 表现 |
|------|------|
| 分类混乱 | 材质/用途/型号混在一起分 |
| 信息失衡 | 要么列太多字段，要么缺关键信息 |
| 不懂概念 | 分不清 Taxonomy vs Term，不知何时建 Post Type |
| 能力盲区 | 不知道 ACF/网站能做什么、不能做什么 |

## 目标

学员用完工具后能产出：
1. 清晰的产品分类结构（Post Type + Taxonomy 设计）
2. 每类产品需要的字段（Field Group 规划）

## 故事

### S1: 初次梳理
学员第一次建站，脑子里只有"我有很多产品"，通过引导式问答，产出结构化的 ACF 规划文档。

### S2: 复杂产品线
学员产品线复杂（多层级组合/定制为主），需要工具帮助拆解，明确哪些用 Taxonomy，哪些用 Repeater，哪些需要独立 Post Type。

### S3: 避免常见错误
学员在梳理过程中，工具能识别并纠正常见错误（如分类维度混乱、信息冗余/不足）。
