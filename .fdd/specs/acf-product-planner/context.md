# 技术决策

## 架构方案

保留现有 think-lab 架构，重点改造：

| 改造点 | 原状 | 目标 |
|--------|------|------|
| System Prompt | 通用思考辅助 | ACF 产品规划专用 |
| 阶段流程 | 通用 5 阶段 | ACF 专用流程（借鉴 5 阶段思维） |
| 最终输出 | 通用思考文档 | ACF 结构化规划文档 |

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 交互方式 | interview 工具 | 复用现有实现，选择题引导 |
| AI 模型 | Claude (thinking) | 现有支持，深度思考能力 |
| 产出格式 | Markdown | 人类可读，可喂给 ACF JSON 生成器 |

## Tradeoff

| 取舍 | 选择 | 代价 |
|------|------|------|
| 行业模板 | 不做 | 学员需要从零梳理，但更通用 |
| Flexible Content | 不引导 | 减少能力覆盖，换取简单性 |
| 中间保存 | 不做 | 必须一次完成，但简化实现 |
| 复杂度分类 | 不强制分类 | AI 根据回答自动判断，不要求学员明确选择类型 |
| 输出确认 | 可选 | 问学员是否需要确认，不强制 |

## 评审澄清

| 质疑 | 结论 |
|------|------|
| JSON 生成器是什么 | 团队已有工具，学员提交 Markdown，团队生成 |
| 一次性完成可行吗 | 接受，学员应一口气梳理完 |
| Taxonomy 概念能教会吗 | 能，用例子引导而非术语 |
| Markdown 格式要严格吗 | 不需要，语义清晰即可 |
