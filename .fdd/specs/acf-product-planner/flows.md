# 核心流程

## 主流程

```
学员输入产品描述 → 产品线概览 → 分类结构设计 → 字段规划 → 结构化文档输出
```

## 阶段详解

### 1. 产品线概览
```
"你有哪几大类产品？" → 识别产品品类 → 初步判断复杂度类型
```

复杂度类型：
- 单品类多规格（螺丝 M3/M4/M5...）
- 多品类少规格（礼品：杯子、笔、本子）
- 多层级组合（整机→模块→零件→规格）
- 定制为主（展示能力，非具体产品）

### 2. 分类结构设计
```
识别分类维度 → 判断 Taxonomy vs Term → 判断是否需要独立 Post Type
```

引导重点：
- 什么是"分类法"（Taxonomy）vs "具体分类"（Term）
- 何时需要新建 Post Type（内容结构差异大时）
- 避免分类维度混乱

### 3. 字段规划
```
按品类确定字段 → 识别通用字段 vs 特有字段 → 判断 Repeater 使用场景
```

引导重点：
- 买家视角：他们最关心什么？
- 信息密度：必填 vs 选填
- 何时用 Repeater（如规格表、多图）
- 不用 Flexible Content（太复杂）

### 4. 输出
```
结构化 Markdown 文档 → 用户拿去喂给 ACF JSON 生成器
```

输出包含：
- Post Type 设计
- Taxonomy 设计（分类法 + 具体分类项）
- Field Group（字段名、类型、说明）
