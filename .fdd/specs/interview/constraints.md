# Constraints

## Scope (做什么)

| 项 | 内容 |
|-----|------|
| 工具定义 | interview tool 的 Zod schema + 类型 |
| 系统提示 | 告诉 AI 何时/如何调用此工具 |
| 示例 UI | 可选，演示用的 React 组件 |

## Non-Goals (不做什么)

| 项 | 原因 |
|-----|------|
| 独立 npm 包 | 先在项目内验证 |
| 通用 UI 组件库 | UI 由消费方实现 |
| 多语言支持 | 问题/选项由 AI 生成，自然支持 |

## 约束条件

| 约束 | 值 |
|------|-----|
| 问题数量 | 1-4 个 |
| 选项数量 | 2-4 个/问题 |
| header 长度 | ≤12 字符 |
| 自动"其他" | 用户始终可输入自定义 |

## 触发条件

AI 应在以下情况调用 interview：
- 需求模糊，存在多种理解
- 需要用户做技术选型决策
- 存在 tradeoff，需用户取舍

## 评审决策

| 问题 | 决策 | 理由 |
|------|------|------|
| AI 判断时机 | 信任 AI | MVP 阶段先验证，观察实际效果 |
| 选项覆盖不足 | 依赖"其他" | 用户自定义输入作为兜底 |
| 字段长度 | 不约束 | 先不管，有问题再迭代 |
